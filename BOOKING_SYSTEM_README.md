# نظام الحجوزات الذكي 📅

## نظرة عامة
صفحة حجوزات متقدمة وذكية للحلاقة مع:

✅ **التوزيع الذكي للحلاقين** - يختار النظام الحلاق الأقل انشغالاً تلقائياً  
✅ **حساب ديناميكي للأوقات** - يحسب الانتظار المتوقع والوقت المتوقع للموعد  
✅ **منع تضارب الحجوزات** - نافذة 15 دقيقة لمنع حجزين في نفس الوقت  
✅ **واجهة سهلة الاستخدام** - بحث فوري وتعديل وحذف  
✅ **حالات متعددة** - معلقة، جاري، اكتمل، ملغى  

## الوصول
الرابط: `http://localhost:5174/bookings` أو من القائمة الجانبية → "الحجوزات"

## الميزات الرئيسية

### 1. عرض الحجوزات
```
┌─────────────────────────────────────┐
│ حجوزات اليوم (3)  │ الحجوزات القادمة (5)
├─────────────────────────────────────┤
│ #1 أحمد محمد                         │
│    أمامك: 0 شخص | الانتظار: 0 دقيقة │
│    الوقت المتوقع: 10:00 صباحاً      │
│    ✂️ الحلاق: علي حسن (تلقائي)       │
│    [تعديل] [حذف] [إنتظار ▼]          │
└─────────────────────────────────────┘
```

### 2. حجز جديد
```
خطوات الحجز:
1. ابحث عن العميل (اسم أو رقم تليفون)
2. حدد التاريخ والوقت
3. اختر مدة الخدمة (اختياري - افتراضي 30 دقيقة)
4. حدد الحلاق (اختياري - سيختار الأقل انشغالاً تلقائياً)
5. أضف نوع الخدمة (اختياري)
6. انقر "إنشاء الحجز"
```

### 3. الحوارات الذكية
- **البحث عن العميل**: يظهر قائمة فوراً أثناء الكتابة
- **التحقق من التوفر**: يمنع الحجز إذا كان الموعد محجوزاً
- **رسائل الخطأ**: واضحة وتساعد المستخدم

## الخوارزميات

### خوارزمية التوزيع الذكي
```
عند الحجز:
1. إذا اختار العميل حلاقاً محدداً → استخدمه
2. إذا اختار "نظام ذكي" (افتراضي):
   - عد الحجوزات لكل حلاق في نفس اليوم
   - اختر الحلاق الأقل عدداً
   - إذا تساووا، اختر الأول

مثال:
اليوم:
  الحلاق أحمد: 2 حجز
  الحلاق علي: 4 حجز  ← الأقل ❌
  الحلاق محمود: 1 حجز ← اختر هنا ✅
```

### خوارزمية حساب الانتظار
```
الحجز رقم 5:
┌─────────────────────┐
│ #1: 30 دقيقة       │
│ #2: 45 دقيقة       │
│ #3: 30 دقيقة       │
│ #4: 40 دقيقة       │
├─────────────────────┤
│ أنت رقم #5:        │
│ الإجمالي: 145 دقيقة + 20 بافر = 165 دقيقة
│ الوقت المتوقع: 2:45 مساءً
└─────────────────────┘
```

### منع التضارب
```
الموعد الجديد: 10:00
نافذة الأمان: ±15 دقيقة = 9:45 إلى 10:15

الفحص: هل توجد حجوزات في هذه النافذة؟
- نعم → ❌ رفض "هذا الموعد محجوز بالفعل"
- لا  → ✅ موافق
```

## عمليات الإدارة

### تعديل الحجز
```
انقر المفتاح الأزرق [✏️ تعديل]
→ تعديل أي بيانات
→ انقر "تحديث الحجز"
```

### حذف الحجز
```
انقر المفتاح الأحمر [🗑️ حذف]
→ تأكيد الحذف
→ تم الحذف من النظام
```

### تغيير الحالة
```
اختر من القائمة المنسدلة:
- في الانتظار: حجز معلق
- جاري: قيد الخدمة
- اكتمل: تمت الخدمة
- ملغى: تم إلغاء الحجز
```

## المتطلبات الأساسية

### جدول في Supabase
```sql
CREATE TABLE bookings (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  clientId UUID NOT NULL REFERENCES clients(id),
  clientName VARCHAR(255) NOT NULL,
  clientPhone VARCHAR(20) NOT NULL,
  barberId UUID REFERENCES barbers(id),
  barberName VARCHAR(255),
  serviceType VARCHAR(255),
  bookingTime TIMESTAMP WITH TIME ZONE NOT NULL,
  duration INTEGER DEFAULT 30,
  queueNumber INTEGER NOT NULL,
  status VARCHAR(50) DEFAULT 'pending',
  notes TEXT,
  createdAt TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
  updatedAt TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);
```

👉 **انظر إلى الملف**: `SETUP_BOOKINGS_TABLE.sql`

## الأخطاء الشائعة وحلولها

### ❌ "الرجاء اختيار عميل أولاً"
**المشكلة**: لم تختر عميلاً من قائمة البحث  
**الحل**: ابحث عن العميل، ثم انقر على اسمه عندما يظهر في القائمة

### ❌ "هذا الموعد محجوز بالفعل"
**المشكلة**: هناك حجز في نفس الموقت (±15 دقيقة)  
**الحل**: اختر وقتاً آخر يختلف بـ 15 دقيقة على الأقل

### ❌ "الرجاء تحديد التاريخ والوقت"
**المشكلة**: لم تملأ حقل التاريخ أو الوقت  
**الحل**: تأكد من ملء كلا الحقلين قبل الحفظ

## نصائح للاستخدام الأمثل

💡 **استخدم النظام الذكي**: اترك اختيار الحلاق فارغاً ليختار النظام الأنسب  
💡 **حدد المدة الصحيحة**: استخدمها لحساب الانتظار بدقة  
💡 **تابع الحالات**: عدّل إلى "جاري" عند بدء الخدمة و"اكتمل" عند الانتهاء  
💡 **استخدم الملاحظات**: أضف متطلبات خاصة العميل في الملاحظات
